SQL Query¶

Using Variance = E[X^2] - E[X]^2

SELECT (SUM(SPEND)/COUNT(SPEND)) AS AVG_SPEND, 

SQRT((SUM(SPEND^2)/COUNT(SPEND)) - (SUM(SPEND)/COUNT(SPEND))^2) AS SD_SPEND

FROM COHORT_DATA

GROUP BY COHORT;

Naive-Bayes¶

  • Uses Bayes' rule to calculate posterior probabilities.
  • Fast and not sensitive to irrelevant features
  • However, it assumes that all variables are independent - likely not true in this case.
  • We build a model using all the features.
In [87]:
nb_model <- naiveBayes(Income_Level ~ ., data = census_data)
nb_test_predict <- predict(nb_model,census_test)

C5.0 Decision Tree¶

  • Fast

  • C5.0 algorithm uses entropy and information gain

  • Categorical variables - Partition by each level and find the best variable with highest information gain.

  • Continuous variables - Get the best cut point to do a binary partition with highest information gain.

  • You can see which attributes are used the most.

In [89]:
C5tree  <- C5.0(census_data[-41],census_data$Income_Level)

pred_tree  <- predict(C5tree, census_test)
